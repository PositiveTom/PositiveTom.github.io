BIOS

Base Input/Output System：基本输入输出系统

实地址模式下的1MB内存布局

  • 地址总线宽度决定了可以访问的内存空间大小。例如16位机的地址总线宽度为20位，地址范围是1MB；32位机的地址总线宽度是20位，地址范围是4GB

CPU眼里，主板上的物理内存（DDR，DDR2）并不是全部的内存。

原因：
计算机中，并不是只有插在主板上的内存条需要通过地址总线访问，还有一些外设需要通过地址总线访问；因此地址总线上会提前预留一些地址空间给这些外设使用。

这也是电脑安装4GB内存条，实际只有3.8GB内存的原因

  • 地址与地址总线的关系
    • 表示地址的一串数字相当于地址总线的输入，相对于参数，和内存条没关系。
  • 图2-1的含义
    • CPU给地址总线提交了一个数字，在地址总线看来，这个数字就是地址。地址分配电路根据此地址的范围，决定在哪个存储介质中分配一个存储单元，最后将此地址和存储单元对应起来。

BIOS之所以加Base的原因

  • BIOS的内存布局
    表2-1中，顶部地址0xF0000～0xFFFFF这64KB的内存就是ROM，里面存储的就是BIOS的代码。
  • BIOS作用：
    • 检测、初始化硬件
      • 如何检测的呢？硬件自己提供了一些初始化功能调用，BIOS直接调用就好！
    • 建立中断向量表（即对硬件的IO操作）
      • 作用：可以通过“int中断号”实现相关的硬件调用
    • 最后一个作用见下一节！！！

由于只有64KB的空间大小，不可能把所有硬件的IO操作实现得面面俱到，精彩的操作是进入保护模式才开始的，所以挑一些重要的、能够保证计算机能运行的那些硬件的基本IO操作。这就是BIOS被称为基本输入输出系统的原因。

如何启动BIOS

  • BIOS是计算机上执行的第一个软件，它是由硬件加载的。
  • 在开机的一瞬间，CPU的cs:ip寄存器被强制初始化为0xF000:0xFFF0；由于开机时处于实地址模式，此时段基址要左移4位之后再与偏移地址相加，得到0xFFFF0这个地址，发送给地址总线，而这个地址就是BIOS的入口地址！！！

注意：

  • BIOS是工作在实地址模式，实地址模式只能访问1MB空间；0xFFFF0距离1MB空间只有16个字节，因此这里的代码只是跳转代码！！！

为什么是0x7c00

  • 程序中以0为索引，是因为人们习惯使用偏移量的概念！！！

BIOS的第三个作用

校验启动盘中位于0盘0道1扇区的内容

为什么不是0盘0道0扇区呢？

硬盘扇区有两种表示方法，0盘0道1扇区是其中一种（CHS方式，即柱面Cylinder 磁头Header 扇区Sector）；另一种LBA不关心

0盘0道1扇区的意义？

  • 0盘：0磁头；一张盘有上下两个盘面，一个盘面对应一个磁头，用Header表示！！！
  • 0道：0柱面；所有盘面上，编号相同的磁道的集合，（磁盘堆叠是一个圆柱体，而这里则是其中的圆柱面（圆环）！！！）
  • 1扇区：把磁道划分的等距小区间，类似扇区！！！扇区的编号是从1开始的，而非0！！！（0开始的方式是LBA方式 ）
  • MBR就在这个扇区！！！

如果0盘0道1扇区的末尾两个字节是0x55和0xaa，BIOS就认为此扇区存在可执行程序（这个程序就是主引导记录MBR）

为什么0盘0道1扇区存储MBR？

  • 因为离BIOS最近，省时！！！否则为了这512字节的程序，BIOS需要查遍整个存储设备的存储单位，去一一比对是否有0x55和0xaa

BIOS为什么会把MBR这段程序加载到 物理内存 为0x7c00 的地址上？

  • 要弄懂上面的问题，首先要知道0x7c00的由来（不是x86开发团队规定的，它属于BIOS开发团队的范畴！！！）
    • 通电开机后，BIOS处理程序开始自检，随后调用BIOS中断0x19h，call int 0x19h。在此中断处理函数中，BIOS要检测计算机含有多少硬盘或者软盘，如果检测到有任何可用的硬盘，就把它的第一个扇区加载到0x7c00！！！
  • BIOS团队之所以这样规定，有以下原因：
    • 首先0x0-0x3ff存放中断向量表，因此MBR不能加载到这里
    • DOS1.0的最小内存32KB，MBR希望不被过早的覆盖，因此要放在32KB的末尾（32KB的16进制是0x8000，2^15次方，15根总线，15个0，4个16进制数，0x8000）
    • MBR本身也是程序，需要用到栈，虽然本身只有512字节，但实际需要大于512字节，因此分配1KB
    • 结合上述3点，因此0x8000-0x400 = 0x7c00

1KB，2^10，10根总线，10个0，3个16进制数，0x400

MBR的放置位置需要满足什么要求？

  • 不能覆盖已有数据
  • 不能过早地被其他数据覆盖
    • 解释：MBR的任务是加载某个程序（一般是内核加载器，很少有加载内核的）到指定位置，并将控制权交给它。（所谓的交控制权就是jmp过去而已，之后MBR的程序就不运行了，也就没有用了，被覆盖也没关系，需要等到MBR真正没有用之后就可以被覆盖了！！！）

MBR

主引导记录程序，咱们能够掌控的第一个程序！！！

MBR特性：

  • 平时所写的程序都是依赖于操作系统，而这个程序是独立于操作系统，可以在裸机上运行！！！
  • MBR大小必须是512字节
  • 扇区末尾两个字节，即第510字节需要是0x55，511字节是0xaa；合在一起是0xaa55，小端字节序（多位数的低位放在较小的地址处，高位放在较大的地址处，与人类阅读习惯相反）

$，$$的含义

  • $：
    • nasm预留的关键字，当前行的地址，起标号的作用；
    • 隐式地表示本行代码的标号，nasm编译器看到了会自动用地址替换
      $$：nasm预留的关键字，本section的地址，起标号的作用
      $ 和 $$不是CPU原生支持的，相当于伪指令！！！对CPU来说是假的

什么是伪指令？

  • 编译器定义，CPU中不存在这个指令，硬要让CPU执行这个指令，会出现“UD（未定义的操作码）”错误
  • 编译器为了开发人员写代码方便而提供的符号
  • 伪指令在编译时，会由编译器转为CPU可识别的东西，指令或者地址等

标号的举例

解释：code_start这个标号被认为是一个地址，地址是”mov ax, 0”这行指令所在的地址，code_start这个标号是假的，cpu不认，到cpu手里之前，nasm会安排数字地址来代替这个标号code_start

关键字vstart


section也是伪指令！！！

MBR简易版程序

首先介绍qemu和bochs

  • bochs
    • 定义：用c++写的开源模拟器，仅可用来模拟x86硬件环境，包括cpu，内存，硬盘，显卡，网卡等。能够用来运行各种os，dos，windows，linux，bsd；还支持许多外部设备，如键盘、鼠标、串口、并口等；还支持多种调试方法，如断点、单步执行、内存监视等。
  • qemu
    • 定义：是一个快速的开源模拟器和虚拟机管理器，能够模拟x86、arm等多种cpu架构。相比bochs，特点在于其快速性，模拟机中的虚拟机能够与物理机的速度相当；也支持各种外部设备，usb设备、串口、网卡等；能够通过gdb、vnc多种调试方法进行调试。

创建虚拟硬盘

  • 软盘映像和硬盘映像的区别： 软盘映像通常比较小，硬盘映像通常比较大；它们只有这一点区别，其磁盘映像的格式没有区别！！！
  • 磁盘映像的格式：
    • raw
    • qcow2
  • 硬盘的类型：可以借鉴
    • flat（整体分配）：整个磁盘映像已经被分配，没有使用稀疏存储。即使磁盘中某些部分没有数据，整个映像也会占用磁盘空间
    • sparse（稀疏存储）：只有数据被写入时，才会占用实际的磁盘空间
    • growing（动态增长）：需要时可以扩展为更大的大小，使得虚拟机运行时可以轻松增加存储空间
  • 在 QEMU 等虚拟化环境中，”img” 通常是一个泛指，表示一般的磁盘映像文件。这个词并不指定具体的磁盘格式，而是包括多种可能的格式，如 raw、qcow2、vmdk 等。

qemu-img命令

  • –object objectdef: 定义 QEMU 用户可创建的对象。这可以是 QEMU 对象的自定义定义，例如密钥或其他配置。

  • -q: 安静模式，只显示错误信息，不显示进度信息。

  • -f fmt: 指定要创建的磁盘映像的格式。例如，可以使用 qcow2 或 raw。如果不指定，默认为 raw 格式。

    • -f raw: 纯粹的二进制格式，没有任何格式化，通常直接与裸机进行交互
    • -f qcow2: 支持增量备份和动态缩放

  • -b backing_file: 指定一个基础映像文件。新创建的映像将作为这个基础映像的后续增量。

  • -F backing_fmt: 指定基础映像的格式。如果不指定，将尝试自动检测。

  • -u: 允许创建不安全的后备链。对于新创建的映像，假定没有任何工作的基础文件。

  • -o options: 以逗号分隔的格式特定选项列表，以 name=value 格式提供。用于配置新创建的映像。

  • filename: 要创建的磁盘映像的文件名。

  • size: 指定磁盘映像的大小。可以使用后缀 k、M、G、T 等表示单位，例如 10G 表示 10GB。

  • 正式创建虚拟硬盘，qcow2格式，60M，安静模式，flat模式
