Linux OS-基于x86

涉及到的专业名词含义：

  • posix（可移植操作系统接口/Portable Operating System Interface）：是IEEE为了在各种UNIX操作系统上运行软件，而定义API的一系列互相关联的标准的总称
  • qmeu：纯软件实现的虚拟化模拟器，几乎可以模拟任何硬件设备（最熟悉的是能够独立运行操作系统的虚拟机，虚拟机认为自己和硬件打交道，实际上是和qemu模拟出来的硬件打交道，qemu将这些指令转译给真正的硬件）
  • nasm: 可以将汇编代码转为可执行的机器代码
  • 引导扇区（boot sector）: 计算机硬盘上的一个特殊区域，它包含引导操作系统的关键信息。当计算机启动，会读取硬盘上的引导扇区，并将控制权转交给这个扇区的代码。
    • 引导扇区的代码 也被称为 固件引导程序
  • tty模式：（tty是终端设备的简称）也叫终端模式，计算机与用户之间进行文本交互的方式
    • 分类：
      • 原始模式（raw mode）：终端设备将用户输入的每个字符都立即发送给程序，而不等待用户按下enter键；通常用于实时应用程序，如游戏。
      • 规范模式（canonical mode）：终端设备会缓冲用户输入，并在用户按下enter键时才将缓冲的一行文本发送给程序。这是默认的终端模式，适用于命令行应用程序和交互式shell。
  • unix: 通用操作系统，有开源版本也有闭源版本
  • linux：开源的unix-like（类unix）操作系统内核
  • ubuntu：基于linux内核特定的发行版（还有fedora，debian，centos）
  • linux内核遵循posix标准，尽量提供符合posix标准的系统调用和接口，这样使得在linux上开发的应用程序可以更容易在其他符合posix标准的系统上运行
  • windows操作系统并不完全遵循posix标准！！！

Linux系统启动的过程

  1. 计算机上电，执行固件引导程序
    • 固件引导程序
      • 存储位置：特殊的EEPROM中
      • 作用：硬件识别，自检，初始化
      • 分类：
        • 称为BIOS，对应的硬盘分区模式是MBR
        • 称为UEFI，更像一个小型操作系统，支持GPT解锁大硬盘，安全启动，对应的硬盘分区模式是GPT
  2. 把控制权交给中间引导程序
    • 以UEFI为例
      • 中间引导程序称为grub2
      • 中间引导程序的存储位置：gpt模式硬盘中的efi系统分区
      • 作用：
        • 加载内核镜像vmlinuz和initramfs到内存
          • vmlinuz: 压缩过的linux系统内核文件
          • initramfs: 临时根文件系统
            • 作用：辅助挂载根文件系统
  3. 内核启动阶段
    • 作用：加载根文件系统，执行根文件系统中的init程序
  4. 执行init程序
    • init程序定义：第一个内核启动的用户级进程
    • 特点：
      • 进程编号是1
      • 是所有其他进程的父进程
      • 系统关闭前不会退出
      • 不是内核的一部分，是独立可更换的程序
    • 不同版本的init程序：
      • sysvinit
      • systemed（目前主流）
        • 作用：根据磁盘配置文件，启动一些系统级服务，也会启动用户配置的服务

Mac

  • loop 无限循环用于等待其他引导操作和等待操作系统加载
  • times： 伪操作符，重复执行某个指令或插入某个数据多次，简化代码的编写
  • 510-($-$$)： 是一个计算表达式
    • 510: 一个常数
    • $：当前位置的符号，表示当前的汇编地址
    • $$：当前节的起始地址
    • $-$$：当前汇编地址-当前节的起始地址；大白话：当前位置相对于当前节的偏移量
  • db 0 ： 伪指令，用0填充字节
  • dw：伪指令
  • dw 0xaa55： 告诉汇编器将将指定数据存储为一个字
  • 使用xxd工具查看上述汇编代码生成的.bin文件如下图所示
    • 55aa代表一个字（2个字节），并且是小端字节序
    • 每一行有8个字，16个字节；总共有32行，32*16=512个字节
